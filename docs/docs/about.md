<h2>What I write about</h2>

<p>This site is where I share practical guidance, deep dives, and lessons learned from building automation and governance solutions for Microsoft 365. Most posts focus on topics such as:</p>

<ul>
    <li>Tenant Configuration Management and Microsoft Graph APIs.</li>
    <li>Microsoft365DSC architecture, troubleshooting, and performance.</li>
    <li>Authentication patterns for Microsoft cloud workloads.</li>
    <li>Configuration drift monitoring and operational automation.</li>
    <li>Real-world governance practices for Microsoft 365 tenants.</li>
</ul>

<h2>Background</h2>

<p>Before joining Microsoft, I was recognized as a Microsoft MVP for my community contributions around PowerShell, Desired State Configuration, and Microsoft 365 automation. My work has always been centered around a simple idea: administrators should be able to manage cloud configuration with the same rigor that developers apply to application code.</p>

<p>I live in Gatineau, Quebec, Canada, and spend much of my time working with customers, engineering teams, and the open-source community to make Microsoft 365 configuration management more predictable, observable, and scalable.</p>
